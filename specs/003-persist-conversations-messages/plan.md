# Implementation Plan: Persist Conversations and Messages

**Branch**: `003-persist-conversations-messages` | **Date**: 2026-06-10 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/003-persist-conversations-messages/spec.md`

## Summary

Add database persistence for CBT coaching chat conversations and messages. The existing in-memory popup chat (history passed via browser, lost on reload) is replaced by a persistent `Conversation`/`Message` record model with RESTful routes and Turbo Stream message delivery. Pundit authorization (not yet installed) enforces per-user ownership. A `Conversations::SendMessageService` orchestrates the atomic conversation+message creation and LLM call. The home-page popup becomes a navigation link to the `/conversations` page.

## Technical Context

**Language/Version**: Ruby 4.0, Rails 8.1  
**Primary Dependencies**: Hotwire (Turbo + Stimulus), ImportMap, Propshaft, SQLite3, RubyLLM  
**New Dependency**: `pundit` gem (not yet in Gemfile — required by Constitution §V)  
**Storage**: SQLite3 (primary)  
**Testing**: RSpec, FactoryBot, Shoulda Matchers, Capybara (system specs)  
**Target Platform**: Web application, single server (Kamal/Docker deployment)  
**Performance Goals**: Chat view loads in under 2 seconds for conversations up to 200 messages; message round-trip (save + LLM + Turbo response) reflects existing LLM latency (no regression)  
**Constraints**: All conversation/message queries scoped to `current_user`; LLM context capped at 50 messages; message content max 10,000 characters  
**Scale/Scope**: Single-tenant per user; personal wellness app; low concurrent request volume

## Hotwire Decision Matrix

| User Interaction | Technique | Rationale |
| ---------------- | --------- | --------- |
| Navigate to `/conversations` | Turbo Drive | Standard navigation; server redirects to most recent conversation |
| Load a specific conversation (`/conversations/:id`) | Turbo Drive | Full page load with complete message history rendered server-side |
| Select past conversation from sidebar | Turbo Drive | Standard link navigation; full page swap |
| Submit first message (creates conversation) | Turbo Drive (redirect) | `create` action redirects to `show` after atomic save |
| Send message to existing conversation | Turbo Stream | Appends user message + assistant reply to message list without full reload |
| Show loading state while LLM responds | Stimulus | `conversation_controller.js` disables input and shows typing indicator during in-flight request |
| Click "New Conversation" | Turbo Drive | Link to `conversations_path`; `index` redirects to blank state or `show` |

## Migration Safety

- [x] All migrations are reversible (`create_table` is reversible by default)
- [x] No mixed DDL and data manipulation in the same migration
- [x] SQLite3 — no `disable_ddl_transaction!` needed; standard index creation is safe
- [x] Foreign key constraints: `conversations.user_id` → `on_delete: :cascade`; `messages.conversation_id` → `on_delete: :cascade`
- [x] Migration order: `create_conversations` before `create_messages` (dependency graph respected)
